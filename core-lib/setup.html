Setting up the Core

Cargo Workspace

We will create a new rust project and make it a workspace to cross-reference multiple crates inside our project.

cargo new quiz-library

Delete the src folder.

Then replace the content of Cargo.toml with our workspace configuration.

[workspace]
members = []

Core Crate

We are going to add crates to our workspace as we go. The first crate is going to be our core crate.

cargo new quiz_core --lib --vcs none

ðŸ“„ Note: We do not need a version control system (vcs) on the new crates since it already initialized one when we ran cargo new quiz-library.

Then add quiz_core as a workspace member to the root Cargo.toml.

[workspace]
members = [
    "quiz_core"
]

Rust Client Crate

I like to separate the client from the library by using different crates. It is a small amount of work yet it makes our project future-proof. For example, adding GUI, it a client dependency and should not be included in our library.

We will do the same as we did for the core crate. Create another crate and add it to the workspace.

cargo new apps/rust_client --bin --vcs none


[workspace]
members = [
    "quiz_core",
    "apps/rust_client"
]

Now we can test it by running:

cargo run --bin rust_client
# Hello, world!

We put the rust client inside an app folder because we will introduce other applications. For iOS, android, and web.

Finally, we need to add and use the library inside the rust client crate we created.

cargo add -p rust_client --path quiz_core/

ðŸ“„ Note: We use -p to specify the package name. In this case, it is rust_client and --path to add a local crate to our project.

use quiz_core::add;

fn main() {
    println!("2 + 2 = {}", add(2, 2));
}

Run again.

cargo run --bin rust_client
# 2 + 2 = 4

The setup is done! Now we can focus on writing the actual library.
